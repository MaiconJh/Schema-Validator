Schema Keywords

Understand supported keywords and how they are interpreted by validators to build powerful schemas.

Structural Keywords

These keywords define the basic structure and shape of your data:

type Required

Specifies the type of data allowed. Valid types: string, number, integer, boolean, array, object, null.

properties

Defines the expected properties of an object. Each key corresponds to a field name, with its value being the schema for that field.

required

An array of property names that must be present in the object. All listed fields must exist for validation to pass.

items

Defines the schema for items in an array. When an array is expected, use this to validate each element against a schema.

Logical Keywords

These keywords allow for complex validation logic and schema composition:

oneOf

Validates that data matches exactly one of the provided schemas. Useful for union types or polymorphic data.

not

Negates a schema - data must NOT match the provided schema. Useful for excluding specific values.

if / then / else

Conditional validation. Apply different schemas based on whether data matches the if condition.

Format and Primitive Constraints

Primitive and format validators enforce low-level correctness before advanced composition rules:

  • format — Validate against common formats (email, uri, date-time, etc.)
  • minimum / maximum — Numeric bounds
  • minLength / maxLength — String length constraints
  • pattern — Regular expression matching
  • enum — Enumeration of allowed values
Design Tip

Start strict on required keys, then add conditional branches incrementally. This approach makes schemas easier to maintain and debug.
