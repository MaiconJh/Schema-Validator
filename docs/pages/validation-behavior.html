Validation Behavior

Learn how validation executes, aggregates errors, and returns final status for reliable data quality assurance.

Evaluation Order

Validation executes in a predictable order to ensure consistent results and meaningful error messages:

Structural Checks First

Type checking and structural validation run before deeper field-level constraints. This ensures basic data shape before validating content.

Field-Level Constraints

Once structural validation passes, field-level constraints like minimum values, patterns, and format are applied.

Error Aggregation Model

Schema Validator provides comprehensive error reporting to help quickly identify and fix validation issues:

  • Multiple Failures — Collects multiple validation failures in a single pass when possible
  • Path Context — Preserves the full JSON path to pinpoint exactly which field failed
  • Script Integration — Exposes errors for downstream scripting decisions in Skript
# Example error output
{
  "valid": false,
  "errors": [
    {
      "path": "user.email",
      "message": "must be a valid email address",
      "keyword": "format"
    },
    {
      "path": "user.age",
      "message": "must be >= 18",
      "keyword": "minimum"
    }
  ]
}
Error Paths

Each error includes the full path from the root of your data, making it easy to locate and fix issues in deeply nested structures.

Conditional Branches

Conditional validators evaluate branch logic and report branch mismatch points clearly:

  • if/then/else — Validates based on conditions
  • oneOf — Ensures exactly one schema matches
  • All conditional errors include context about which branch failed
Performance Note

Complex conditional schemas may take longer to validate. Keep schemas as simple as possible while meeting your requirements.
