Installation

Install and bootstrap Schema Validator with project-friendly defaults. Get up and running in minutes.

Installation Flow

Follow these steps to install and configure Schema Validator on your server:

# Step 1: Add plugin artifact to your server's plugins folder
# Download the latest release from the releases page

# Step 2: Start/restart your server to generate default configuration files
# The plugin will create default schema directories on first load

# Step 3: Confirm schema directories exist
/your-server/plugins/schema-validator/schemas/
/your-server/plugins/schema-validator/examples/

# Step 4: Restart after making initial configuration changes

System Requirements

  • Java Runtime — Java 8 or higher (Java 17 recommended for best performance)
  • Skript — A compatible Skript plugin installed on your server
  • Server Platform — Any server platform supporting Skript plugins (Spigot, Paper, etc.)
  • Storage — At least 50MB for the plugin and example schemas

Verify Installation

After installation, verify everything is working correctly:

# Check runtime logs for plugin load status
[SchemaValidator] ✓ Schema Validator v1.0.0 loaded successfully
[SchemaValidator] ✓ Registered 5 schema keywords
[SchemaValidator] ✓ Schema directory: ./plugins/schema-validator/schemas

# Confirm schema directory paths are present
ls ./plugins/schema-validator/
Success

When you see the success message in your server logs, Schema Validator is ready to use!

Troubleshooting

Plugin does not load

  • Confirm version compatibility with your server software
  • Inspect server logs for dependency issues or missing requirements
  • Ensure Skript is installed and loaded before Schema Validator

Schemas are not discovered

  • Confirm schema path settings in configuration file
  • Verify file permissions allow reading schema files
  • Ensure schema files have correct extensions (.json, .yaml, .yml)
Common Issue

If schemas aren't loading, check that your schema files are valid JSON/YAML. Invalid syntax will prevent schema registration.
