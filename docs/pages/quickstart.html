Quickstart

Execute your first successful and failing validation in minutes. Get hands-on experience with Schema Validator.

Your First Validation

Follow these steps to execute your first schema validation:

  1. Create a minimal schema — Define the expected structure of your data
  2. Create matching YAML/JSON data — Prepare input data that conforms to your schema
  3. Trigger validation through integration syntax — Use Skript commands to validate
  4. Inspect result status and errors — Review validation output for success or failure details
Pro Tip

Start with simple schemas to understand the basics before moving to complex nested structures.

Minimal Schema Pattern

Here's a basic schema that requires an id field of type string:

{
  "type": "object",
  "required": ["id"],
  "properties": {
    "id": { "type": "string" }
  }
}

Schema Breakdown

  • "type": "object" — Defines the root as an object
  • "required": ["id"] — The id field must be present
  • "properties" — Defines the expected structure of each field

Valid Data Example

This data passes validation because it has the required id field with a string value:

{
  "id": "player-123",
  "name": "John Doe"
}

Expected Outcomes

Success Case

When data contains all required fields with valid types, validation returns a success result with no errors.

Failure Case

When validation fails, the result includes actionable error messages pointing to specific field issues.

Error Messages

Error messages include the JSON path to the problematic field, making debugging straightforward.
