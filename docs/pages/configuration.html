Configuration

Configure schema directories, runtime behavior, and safe defaults to customize Schema Validator for your project.

Configuration Priorities

When configuring Schema Validator, focus on these key areas for optimal performance:

Schema Path Configuration

Define where schemas are discovered and loaded. Configure explicit paths for better control over schema management.

Error Verbosity

Configure error output levels - use concise end-user messages or detailed operator logs for debugging.

Reload Strategy

Prefer explicit reload operations in production for stability, or automatic reloading during development.

Common Options

Schema Path Configuration

Define where schemas are discovered and reloaded. The plugin scans these directories for schema files on startup and when triggered.

schema:
  path: ./plugins/schema-validator/schemas
  watch: false
  reloadOnChange: false

Error Verbosity

Control how detailed error messages are. Choose between user-friendly messages or developer-focused debugging output.

logging:
  level: info  # trace, debug, info, warn, error
  verboseErrors: false
  includePaths: true

Reload Strategy

Configure when schemas should be reloaded to balance performance and development workflow.

runtime:
  autoReload: false  # Recommended for production
  cacheSchemas: true
  maxCacheSize: 100
Development vs Production

Enable auto-reload during development for faster iteration. Disable it in production for better performance and stability.
