Architecture

Component-level architecture for loading, dispatching, and validating data. Understand how the pieces work together.

Main Components

Schema Validator consists of four core components that work together to provide comprehensive data validation:

File Schema Loader

Responsible for loading schemas from the file system. Handles various file formats (JSON, YAML) and manages schema file discovery and caching.

Schema Registry

Maintains a registry of all loaded schemas. Provides schema lookup, versioning, and lifecycle management capabilities.

Validation Service

Core validation engine that executes validation logic. Dispatches validation requests to appropriate validators based on schema keywords.

Integration Bridge

Connects the validation system with Skript. Exposes commands, expressions, and events for seamless workflow integration.

Validation Pipeline

Each validation request follows a five-step pipeline from schema loading to error reporting.

  1. Schema Registration at Startup — Schemas are loaded and registered when the plugin initializes
  2. Runtime Validation Request — A validation request is triggered through Skript integration
  3. Schema and Data Resolution — The appropriate schema is retrieved and data is parsed
  4. Keyword Validator Dispatch — The validation service dispatches to specific keyword validators
  5. Result and Error Propagation — Validation results are collected and returned with error details

Operational Guidance

Follow these best practices for maintainable implementations:

  • Use Modular Schemas — Break complex schemas into reusable components
  • Keep Keyword Logic Isolated — Separate validation concerns for easier maintenance
  • Cache Frequently Used Schemas — Improve performance by caching validated schemas
  • Monitor Validation Performance — Track validation times for optimization opportunities
Best Practice

Modular schemas with clear separation of concerns are easier to test, debug, and extend over time.
