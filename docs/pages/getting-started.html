Getting started

Understand prerequisites, core concepts, and terminology before installation to maximize your productivity.

Core Concepts

Familiarize yourself with these fundamental concepts that power Schema Validator:

Schema Registration

Loads and resolves schemas from file system, enabling validation of structured data against defined rules.

Validation Dispatcher

Routes data nodes to the appropriate validators based on schema type and keyword specifications.

Integration Layer

Exposes commands and results to Skript, enabling seamless workflow automation with validation feedback.

Prerequisites

Before you begin, ensure you have the following:

  • Compatible Java Runtime — Java 8 or higher is required for the validator to function.
  • Plugin Configuration Access — Administrative access to modify plugin settings.
  • Schema Files — Access to JSON/YAML schema files for your validation needs.
  • Familiarity with JSON/YAML — Basic understanding of structured data formats.
Important

Schema Validator requires a Skript-compatible plugin environment. Ensure your server supports Skript plugins before proceeding.

Terminology

Schema

A rule set defining allowed data patterns. Schemas specify what fields are required, what types are valid, and what constraints apply.

Data File

A JSON/YAML input file that is validated against a schema to ensure it conforms to the expected structure and types.

Validation Result

A structured output containing success/failure status along with detailed error information including field paths and error messages.
