Examples

Practical examples for common schema patterns and debugging workflows to accelerate your development.

Basic Object Validation

Use object schemas with required keys and type checks for profile data:

{
  "type": "object",
  "required": ["id", "name", "email"],
  "properties": {
    "id": { "type": "string" },
    "name": { "type": "string", "minLength": 1 },
    "email": { "type": "string", "format": "email" },
    "age": { "type": "integer", "minimum": 0 }
  }
}

Nested Address Schemas

Compose nested objects to validate complex game entity structures:

{
  "type": "object",
  "required": ["player"],
  "properties": {
    "player": {
      "type": "object",
      "required": ["id", "inventory"],
      "properties": {
        "id": { "type": "string" },
        "inventory": {
          "type": "array",
          "items": {
            "type": "object",
            "required": ["itemId", "quantity"],
            "properties": {
              "itemId": { "type": "string" },
              "quantity": { "type": "integer", "minimum": 1 }
            }
          }
        }
      }
    }
  }
}

Conditional Examples

Use if / then / else for context-aware constraints:

{
  "type": "object",
  "properties": {
    "membership": { "type": "string", "enum": ["free", "premium"] }
  },
  "if": {
    "properties": { "membership": { "const": "premium" } }
  },
  "then": {
    "required": ["subscriptionDate"],
    "properties": {
      "subscriptionDate": { "type": "string", "format": "date-time" }
    }
  },
  "else": {
    "properties": {
      "adsEnabled": { "type": "boolean" }
    }
  }
}

Debug Loop

Follow this iterative workflow when debugging validation issues:

Debug Workflow

  1. Run validation — Execute validation against your data file
  2. Inspect error list — Review all returned errors with their paths
  3. Fix schema or input — Update either the schema or data to resolve issues
  4. Re-run and compare — Validate again and compare results
Pro Tip

Start with simple schemas and incrementally add complexity. This makes debugging much easier than building complex schemas all at once.
